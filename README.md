
# Selenium Course Final Project

Итоговый проект курса **«Автоматизация тестирования с помощью Selenium и Python»**  

Тестирование [интернет-магазина](https://pythonanywhere.com)


## О проекте

Проект представляет собой набор автотестов на основе паттерна **Page Object Model (POM)**, тестируется демонстрационный интернет-магазин книг, покрыты ключевые пользовательские сценарии: навигация, авторизация, регистрация, добавление товара в корзину и проверка состояния корзины


##  Технологии

- язык программирования **Python**
- управление браузером **Selenium WebDriver**
- фреймворк для тестирования **PyTest**
- браузер для запуска тестов **Chrome**


## Структура проекта

```
selenium_course_final_project/
```

##  Описание структуры проекта

### `conftest.py` (Конфигурация тестового окружения)

- Добавлен кастомный параметр командной строки `--language` для запуска тестов на разных языках интерфейса (например: `en`, `ru`, `fr`, `es` и др.)
- Определена фикстура `browser` со `scope="function"`, создаёт и закрывает экземпляр **Chrome WebDriver** для каждого теста
- При старте браузера автоматически применяются языковые настройки через `intl.accept_languages`


### `pages/locators.py` (CSS-селекторы элементов)

Централизованное хранилище всех CSS-селекторов, разбитых по классам страниц:

| Класс локаторов       | Описание                                                                |
| --------------------- | ----------------------------------------------------------------------- |
| `BasePageLocators`    | Кнопка логина/регистарцию, иконка пользователя, кнопка корзины          |
| `LoginPageLocators`   | Формы входа и регистрации, поля: email/пароль, кнопка регистрации       |
| `ProductPageLocators` | Кнопка добавления в корзину, название товара, цена, сообщения об успехе |
| `BasketPageLocators`  | Сообщение о пустой корзине, список товаров в корзине                    |


### `pages/base_page.py` (Базовый класс страницы)

Родительский класс для всех Page Object, реализует универсальные методы, используемые на любой странице:

| Метод                                        | Описание                                                                         |
| -------------------------------------------- | -------------------------------------------------------------------------------- |
| `open()`                                     | Открывает страницу по URL                                                        |
| `is_element_present(how, what)`              | Проверяет наличие элемента на странице                                           |
| `is_not_element_present(how, what, timeout)` | Убеждается, что элемент отсутствует в течение заданного времени (явное ожидание) |
| `is_disappeared(how, what, timeout)`         | Ждёт, пока элемент исчезнет со страницы                                          |
| `go_to_login_page()`                         | Кликает по ссылке Войти и переходит на страницу входа                            |
| `go_to_basket_page()`                        | Кликает по иконке корзины и переходит в корзину                                  |
| `should_be_login_link()`                     | Ассертит наличие ссылки на логин                                                 |
| `should_be_authorized_user()`                | Проверяет, что пользователь авторизован (иконка профиля)                         |
| `solve_quiz_and_get_code()`                  | Решает математическую задачу из JS-алерта и принимает промо-код                  |


### `pages/main_page.py` (Главная страница)

Класс `MainPage` наследуется от `BasePage`, инициализирует объект главной страницы магазина, в текущей версии использует методы базового класса без расширений


### `pages/login_page.py` (Страница авторизации и регистрации)

Класс `LoginPage` с методами:

| Метод                                | Описание                                                                                       |
| ------------------------------------ | ---------------------------------------------------------------------------------------------- |
| `should_be_login_page()`             | Комплексная проверка: URL содержит слово "login", присутствуют обе формы: логина и регистрации |
| `should_be_login_url()`              | Проверяет, что текущий URL содержит слово login                                                |
| `should_be_login_form()`             | Проверяет наличие формы входа                                                                  |
| `should_be_register_form()`          | Проверяет наличие формы регистрации                                                            |
| `register_new_user(email, password)` | Заполняет и отправляет форму регистрации нового пользователя                                   |


### `pages/product_page.py` (Страница товара)

Класс `ProductPage` с методами для тестирования добавления товара в корзину:

| Метод                                     | Описание                                                                              |
| ----------------------------------------- | ------------------------------------------------------------------------------------- |
| `add_to_basket()`                         | Нажимает кнопку Add to basket, если появляется JS-алерт с квизом, то решает его       |
| `should_be_product_added_message()`       | Проверяет, что в сообщении об успехе название товара совпадает с тем, что на странице |
| `should_be_basket_total_message()`        | Проверяет, что цена товара совпадает со стоимостью корзины в сообщении                |
| `should_not_be_success_message()`         | Негативная проверка: сообщение об успехе должно отсутствовать                         |
| `should_be_success_message_disappeared()` | Негативная проверка: сообщение об успехе должно исчезнуть                             |


### `pages/basket_page.py` (Страница корзины)

Класс `BasketPage` с методами:

| Метод                                | Описание                                            |
| ------------------------------------ | --------------------------------------------------- |
| `should_be_empty_basket()`           | Проверяет наличие сообщения о пустой корзине        |
| `should_not_be_products_in_basket()` | Проверяет, что список товаров в корзине отсутствует |


##  Тест-кейсы

### `test_main_page.py` (Тесты главной страницы)

| Тест                               | Описание                                                  |
| ---------------------------------- | --------------------------------------------------------- |
| `test_guest_can_go_to_login_page`  | Гость может перейти на страницу логина с главной страницы |
| `test_guest_should_see_login_link` | На главной странице присутствует ссылка для входа         |


### `test_product_page.py` (Тесты страницы товара)

#### Добавление товара в корзину (параметризованный тест)

| Тест                                   | Описание                        |
| -------------------------------------- | ------------------------------- |
| `test_guest_can_add_product_to_basket` | Гость добавляет товар в корзину |

#### Негативные проверки

| Тест                                  | Описание                                                                                  |
| ------------------------------------- | ----------------------------------------------------------------------------------------- |
| `test_guest_cant_see_success_message` | На странице товара нет сообщения об успехе до добавления в корзину                        |
| `guest_cant_see_success_message`      | После добавления товара сообщение об успехе не появляется (ожидаемо упадёт xfail)         |
| `message_disappeared`                 | Сообщение об успехе исчезает после добавления товара (ожидаемо упадёт xfail)              |

#### Навигация со страницы товара

| Тест                               | Описание                                                  |
| ---------------------------------- | --------------------------------------------------------- |
| `should_see_login_link`            | На странице товара есть ссылка для входа                  |
| `guest_can_go_to_login_page`       | Гость может перейти на страницу логина со страницы товара |
| `guest_cant_see_product_in_basket` | При открытии корзины со страницы товара корзина пуста     |

#### Тесты авторизованного пользователя (`UserAddToBasket`)

Класс использует autouse-фикстуру `setup`, которая перед каждым тестом автоматически регистрирует нового пользователя с уникальным email и проверяет успешную авторизацию

| Тест                                  | Описание                                                                      |
| ------------------------------------- | ----------------------------------------------------------------------------- |
| `test_user_cant_see_success_message`  | Авторизованный пользователь не видит сообщение об успехе до добавления товара |
| `test_user_can_add_product_to_basket` | Авторизованный пользователь добавляет товар, проверяется название и цена      |


### `pytest.ini` — Конфигурация pytest

Кастомный маркер: `need_review`: Tests that need review


##  Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/therealyourvanechka/selenium_course_final_project.git
cd selenium_course_final_project
```

### 2. Установка зависимостей

```bash
pip install selenium pytest
```

> Убедитесь, что установлен **ChromeDriver**, соответствующий версии вашего браузера Chrome

### 3. Запуск тестов

**Все тесты на английском языке (по умолчанию):**

```bash
pytest -v .
```

**Все тесты на русском языке:**

```bash
pytest -v --language=ru .
```

**Только тесты, требующие ревью:**

```bash
pytest -v -m need_review .
```

**Только тесты главной страницы:**

```bash
pytest -v test_main_page.py
```

**Только тесты страницы товара:**

```bash
pytest -v test_product_page.py
```
